# Day003 Lesson

Topic: IP 承载网入门：运营商为什么需要一张“送数据的公路网”

Status: Ready - Beginner Friendly

## 今天学习目标

从今天开始，整体课程先进入 **IP 承载网**。你可以先把它理解成：

> 运营商内部用来把用户数据从接入点送到业务出口、互联网或内容平台的一张“数据公路网”。

今天只解决一个主问题：

```text
用户家里的数据离开家庭网络以后，运营商内部靠什么把它继续送到目标方向？
```

学完以后，你应该能用自己的话解释：

- IP 承载网不是 Wi-Fi，也不是光纤本身，而是运营商内部负责转发 IP 数据的网络。
- IP 包像写着目的地址的快递，路由器像高速路口的分拣站。
- “承载”不是创造业务，而是把宽带、专线、 IPTV、云访问等业务流量可靠送过去。
- IP 承载网会连接接入网、光传送网、互联网出口、数据中心和业务平台。
- 今天只打开 IP 承载网的大门；光传送和光接入后面会专项讲。

## 今天先懂这些词

| 术语 | 通俗解释 | 正式含义 | 例子 | 工程意义 |
| --- | --- | --- | --- | --- |
| IP | 快递单上的目的地编号 | Internet Protocol，用于在三层网络中标识主机和转发数据包 | `192.168.1.10`、`203.0.113.10` | 路由、寻址、上网排障都离不开 IP |
| IP 包 Packet | 一件写着目的 IP 的快递 | 网络层数据单元，包含源 IP、目的 IP 等字段 | 电脑访问网站时产生很多 IP 包 | 工程师排障时要看包从哪里来、到哪里去 |
| 承载 Bearer | 负责把业务流量送过去的公路 | 网络为上层业务提供传输和转发能力 | 家庭宽带流量被运营商网络承载到互联网出口 | 运营商要保证带宽、时延、可靠性和隔离 |
| IP 承载网 | 运营商内部送 IP 数据的公路网 | 由路由器、交换机、BNG/BRAS、链路和路由协议组成的 IP 转发网络 | 用户宽带流量从 OLT 上来后进入 BNG/BRAS，再进入骨干/出口 | 固网业务能否稳定、快速、可运营，核心靠它 |
| 路由器 Router | 高速路口的分拣站 | 根据路由表把 IP 包转发到下一跳的三层设备 | 汇聚路由器、核心路由器、BNG/BRAS | 决定流量路径、故障范围和扩容方式 |
| 下一跳 Next Hop | 下一站先交给谁 | 当前设备转发到目标网段时选择的下一个三层设备 | 家庭电脑出网先交给默认网关 | 排障时常问“下一跳通不通、路由对不对” |
| BNG/BRAS | 宽带用户进入运营商 IP 网络的服务门口 | Broadband Network Gateway / Broadband Remote Access Server，负责用户接入、认证、地址、策略等 | 家庭宽带拨号或 IPoE 用户会话通常在这里管理 | 它连接接入侧用户和运营商 IP 承载网 |

## 1. 先用一个生活例子理解 IP 承载网

你从家里寄一个包裹到外地朋友手上，过程大概是：

```text
家门口 -> 小区驿站 -> 区域分拨中心 -> 省际干线 -> 目的城市分拨中心 -> 朋友家
```

网络里也类似：

```text
电脑/手机
  -> 家庭 CPE/ONT
  -> OLT/接入网
  -> BNG/BRAS
  -> 汇聚/核心 IP 承载网
  -> 互联网出口或内容平台
```

今天重点是中间的 **IP 承载网**。它不像家庭 Wi-Fi 那样离你很近，也不像光纤那样是一根线。它更像运营商内部的一组“数据公路、路口和规则”。

延伸学习：

- [RFC 791: Internet Protocol](https://www.rfc-editor.org/rfc/rfc791)  
  为什么推荐：这是 IPv4 的基础标准入口，可以帮助你确认 IP 的核心作用是寻址和转发。  
  今天读到什么程度：只看 Introduction，知道 IP 负责把分组从源送到目的即可。

## 2. “承载”到底是什么意思

承载不是业务本身。

家庭宽带、企业专线、IPTV、云专线、手机回传，这些是业务或业务场景。承载网做的是：

```text
把这些业务产生的数据，按照规则、带宽、优先级和可靠性要求送过去。
```

类比：

- 电商平台是业务。
- 快递网络是承载。
- 商品不是快递网络生产的，但快递网络决定它能不能按时送达。

运营商工程里，承载网要关注：

- 带宽够不够：高峰期会不会拥塞。
- 时延高不高：游戏、视频会议是否卡顿。
- 路径稳不稳：设备或链路故障后能否绕行。
- 业务能否隔离：家庭宽带、政企专线、 IPTV 是否互不影响。
- 是否好运营：能否统计用户、套餐、流量、故障和质量。

## 3. IP 包和路由器：今天只先懂“按目的地转发”

IP 包里有很多字段，今天先抓住两个：

```text
源 IP：谁发出的
目的 IP：要送到哪里
```

路由器收到 IP 包后，会看目的 IP，然后查自己的路由表：

```text
目的地是 8.8.8.8
我应该从哪个接口发出去？
下一跳是谁？
```

这就是 IP 承载网的基础动作：**查表转发**。

今天不展开 OSPF、IS-IS、BGP、MPLS、SRv6。你只需要先知道：

```text
路由协议负责让路由器学到路；
路由表负责指导每一个 IP 包往哪走。
```

后面讲协议时，会详细打开“谁给谁发报文、报文里有什么、路由怎么被算出来”。

延伸学习：

- [RFC 1812: Requirements for IP Version 4 Routers](https://www.rfc-editor.org/rfc/rfc1812)  
  为什么推荐：这是 IPv4 路由器行为的经典标准，能确认路由器不是“随便转发”，而是有明确要求。  
  今天读到什么程度：只读 2.2.1 和 2.2.2 附近标题，知道路由器连接多个 IP 网络并转发数据即可。

## 4. IP 承载网里常见的设备角色

今天不要求背型号，但要开始建立“设备地图”。

### BNG/BRAS

它像宽带用户进入运营商网络的大门。

常见职责：

- 用户认证：确认这个用户能不能上网。
- 地址分配或地址管理：配合 DHCP、PPPoE 或 IPoE。
- 会话管理：用户上线后形成一条会话。
- 策略控制：套餐速率、QoS、访问控制、计费相关信息。

华为场景里，BNG/BRAS 功能可能由运营商级路由器或专用宽带网关设备承载。具体产品会随运营商建设年代和网络规模不同而不同。

### 汇聚路由器

它像区域分拨中心。

多个接入区域或多个 OLT/BNG 上来的流量，会汇聚到更大的路由节点，然后继续送往核心或出口。

### 核心路由器

它像省际或全国主干高速。

核心路由器关注高容量、高可靠、快速收敛和跨区域调度。

### 业务边界/出口路由器

它像城市对外高速口。

互联网出口、云连接、内容平台、 CDN、政企专线出口等，通常会在边界位置和其他网络互联。

设备例子：

- 华为 NetEngine 系列常出现在运营商 IP 承载、城域、核心、边缘等场景。
- 这里把它当作真实工程中的设备族例子，不要求记具体配置命令。

## 5. IP 承载网和光传送、光接入是什么关系

今天只点到为止。

```text
光接入网：解决光纤怎么到用户家，比如 OLT、ODN、ONT。
光传送网：解决大容量流量怎么在城市、省份、数据中心之间用光网络传送，比如 OTN/WDM。
IP 承载网：解决 IP 包进来以后怎么按路由和策略送到目标方向。
```

简单链路：

```text
用户家庭
  -> 光接入网
  -> IP 承载网
  -> 光传送网承载的长距离链路
  -> 互联网/云/内容平台
```

现实网络里它们会交织在一起，但学习时先分层理解。

## 6. 工程视角：不同岗位怎么看 IP 承载网

规划：

- 用户规模有多少，峰值带宽要多大。
- BNG/BRAS 放在哪里，汇聚节点怎么分区。
- 地址怎么规划，哪些业务需要隔离。

建设：

- 设备上架、链路开通、端口规划、VLAN/IP 地址配置。
- 与光传送、接入侧、互联网出口联调。

维护：

- 看接口状态、路由表、用户会话、链路误码、丢包、时延。
- 华为 VRP 风格常见观察方向：

```text
display interface brief
display ip routing-table
display arp
display access-user
display logbuffer
```

运营：

- 套餐速率是否生效。
- 用户在线数、流量趋势、投诉区域是否集中。

优化：

- 调整链路容量、路由路径、QoS 策略。
- 减少拥塞、绕路、单点故障。

营销：

- 某区域是否具备千兆/万兆宽带承载能力。
- 是否能支撑低时延游戏、高清视频、企业专线、云办公等卖点。

## Knowledge Review

这里复习 Day002 低分或容易混淆点，但不把它们变成今天主课。

- DNS Resolution Path：DNS 负责把域名查成 IP；它不负责选择 IP 承载网中的路径。
- Home Gateway vs ONT Role：家庭侧默认网关/NAT 可能在路由器或路由型 ONT；普通桥接 ONT 主要做光接入转换。
- BNG/BRAS Session Timing：用户会话通常在接入认证阶段建立；访问每个网站时不需要重新认证一次。

IP 承载网就是运营商内部让 IP 数据“按地址、按策略、按质量要求继续往前走”的网络；今天先懂它是什么，后面再逐步拆路由、协议、设备和工程。
