# Day003 Lesson

Topic: 以太网、MAC、ARP 和默认网关选择

Status: Ready - Beginner Friendly

## 今天学习目标

Day001 你建立了固网大地图。Day002 你沿着“打开一个网站”分清了 DHCP、DNS、默认网关、NAT、ONT、BNG/BRAS 的职责。

今天只往下钻一层：当电脑已经知道目标 IP 以后，它在本地网络里到底先把数据交给谁？

学完以后，你应该能用自己的话解释：

- 以太网像家庭或小区内部的投递方式，不是整个互联网。
- MAC 地址像本地网络里的“设备门牌”。
- 交换机/CPE 的交换功能根据 MAC 地址在本地转发帧。
- ARP 用来把“本地要交给谁的 IP”查成 MAC 地址。
- 访问同一家庭网段的设备时，电脑 ARP 查询的是对方设备。
- 访问外网时，电脑 ARP 查询的是默认网关，不是远端网站服务器。
- DNS 查询返回网站 IP；ARP 解决本地下一跳 MAC。这两个不要混在一起。

## 今天先懂这些词

| 术语 | 通俗解释 | 正式含义 | 家庭例子 | 工程意义 |
| --- | --- | --- | --- | --- |
| 以太网 Ethernet | 家里或小区内部“怎么把一份东西递给隔壁设备”的规则 | 一组局域网链路层技术，常见于有线局域网、运营商接入和汇聚网络 | 电脑用网线接到路由器 LAN 口 | 固网里大量接入、汇聚、家庭 LAN 都离不开以太网帧和 MAC |
| 帧 Frame | 本地链路上投递的一封小信封 | 二层数据单元，包含源/目的 MAC 等链路层信息 | 家里电脑给手机传文件时，CPE 在本地转发一帧帧数据 | 抓包和交换排障时，要区分“帧”和三层 IP 包 |
| MAC 地址 | 本地网络里的设备门牌 | 网卡在二层网络中使用的链路层地址，常见以太网 MAC 为 48 bit | 手机 Wi-Fi 网卡有自己的 MAC，电脑网卡也有自己的 MAC | 交换机靠 MAC 学习和转发；ARP 结果也是 IP 到 MAC 的映射 |
| 交换 Switch | 小区内部前台，知道哪台设备在哪个门口 | 二层设备根据 MAC 地址学习表转发以太网帧 | 家里电脑访问家里 NAS，不需要出互联网 | 判断家庭内互访、同 VLAN 转发、二层环路和 MAC 学习问题的基础 |
| ARP | 问“这个本地 IP 对应哪个 MAC” | Address Resolution Protocol，把 IPv4 地址解析为本地链路可用的 MAC 地址 | 电脑问：`192.168.1.1` 这个网关是谁的门牌？ | 没有正确 ARP，终端可能知道 IP 路径，却发不出本地第一跳 |
| ARP 缓存 | 临时通讯录 | 主机保存一段时间的 IP 到 MAC 映射表 | 电脑刚问过网关 MAC，短时间内不用每次都重新问 | 排障时可用 `arp -a`、`ip neigh`、`display arp` 看证据 |
| 默认网关 | 出家门的大门 | 访问非本地网段时交给的下一跳三层设备 | 家里电脑访问公网网站，先交给 CPE 的 LAN 地址 | 外网访问的 ARP 目标通常是网关 MAC，不是网站 MAC |
| 广播 Broadcast | 在本地网络里喊一嗓子 | 二层广播帧会被同一广播域内设备接收 | “谁是 `192.168.1.1`？请告诉我” | ARP 请求用广播；广播域大小影响性能、故障范围和 VLAN 设计 |

## 1. 先把“本地投递”和“远方投递”分开

家庭生活类比：

你住在一个小区。你要把文件交给同一栋楼的邻居，可以直接在小区内部找门牌；你要寄给外省朋友，就先把信交给小区门口的快递点。

网络里也是这样：

```text
同一家庭网段：
电脑 -> CPE 的交换功能 -> 手机/NAS

访问外网：
电脑 -> 默认网关 CPE -> ONT -> OLT -> 汇聚 -> BNG/BRAS -> Internet
```

今天的关键不是背层级，而是先会问一个问题：

```text
这个目标 IP 在不在我的本地网段？
```

如果在本地网段，终端要找目标设备的 MAC。

如果不在本地网段，终端要找默认网关的 MAC。

延伸学习：

- [IEEE 802.3-2022: IEEE Standard for Ethernet](https://standards.ieee.org/ieee/802.3/10422/)  
  为什么推荐：IEEE 802.3 是以太网标准入口，能帮助你知道 Ethernet 是标准化的局域网技术，不是某个厂商的叫法。  
  今天读到什么程度：只看页面标题和简介，知道 Ethernet 涉及 MAC、不同速率和不同介质即可。

## 2. MAC 地址：本地网络里的门牌，不是公网地址

家庭生活类比：

IP 地址像“你要去哪个城市、哪个小区”；MAC 地址更像“到了这个小区以后，具体送到哪个门口”。

在家庭局域网里，电脑、手机、CPE 的 LAN 口都有自己的 MAC 地址。电脑要在这个本地网络里发一帧数据时，以太网帧里会有：

```text
源 MAC：我这块网卡的门牌
目的 MAC：这次本地投递要交给谁
```

正式定义：MAC 地址是二层链路层地址，用于同一二层网络或同一链路范围内的帧投递。常见以太网 MAC 地址长度是 48 bit，通常写成类似 `aa:bb:cc:dd:ee:ff` 的形式。

工程意义：

- 交换机学习“某个 MAC 从哪个端口来”，再按目的 MAC 转发。
- 同一家庭网络里，设备互访通常依赖 MAC 层转发。
- 访问外网时，远端网站服务器的 MAC 不会一路出现在你家电脑的本地以太网帧里；电脑本地帧的目的 MAC 通常是默认网关。

## 3. 交换：家庭内部设备互访不一定去互联网

家庭生活类比：

你家电脑把照片传给家里手机，不需要把照片先寄到外省再寄回来。只要两个设备在同一个家庭网络里，CPE 的交换功能就像家里的内部前台，把数据从一个本地端口转到另一个本地端口。

简单路径：

```text
电脑 192.168.1.10
  -> CPE 内部交换
  -> 手机 192.168.1.20
```

正式定义：交换是二层转发行为。交换设备根据收到帧的源 MAC 学习地址表，再根据目的 MAC 决定从哪个端口转发。

工程意义：

- 如果同一家庭网络内设备互访失败，未必是运营商链路问题。
- 排障时要先确认设备是否在同一网段、同一 Wi-Fi/LAN、同一 VLAN。
- 华为 VRP 风格排障中，二层证据常围绕 MAC 地址表、接口状态、VLAN 和 ARP 进行，例如：

```text
display mac-address
display interface brief
display vlan
display arp
```

这些命令今天只用于建立“看证据”的方向，不要求你在真实设备上执行。

## 4. ARP：知道 IP 后，还要知道本地下一跳 MAC

家庭生活类比：

你知道邻居叫“张三”，也知道他住在本小区，但你不知道他具体门牌。于是你在小区群里问：

```text
谁是张三？请告诉我你的门牌。
```

张三回复：

```text
我是张三，我的门牌是 3 栋 501。
```

ARP 做的事情类似。终端知道某个本地 IP，但还不知道本地投递用哪个 MAC，就发 ARP 请求。

同网段访问示例：

```text
电脑：192.168.1.10
手机：192.168.1.20

电脑要访问手机：
ARP 请求：谁是 192.168.1.20？
手机回复：192.168.1.20 的 MAC 是 xx:xx:xx:xx:xx:20
电脑把帧发给手机的 MAC
```

正式定义：ARP 是 Address Resolution Protocol，用于把 IPv4 地址解析成同一本地链路上可投递的链路层地址，例如以太网 MAC 地址。

工程意义：

- ARP 只解决本地链路的下一跳 MAC，不负责查域名。
- ARP 请求通常是广播，ARP 回复通常是单播。
- ARP 结果会进入缓存，所以不是每个包都重新广播查询。
- 排障时，如果 IP 配置看似正确但无法到达本地网关，可以检查 ARP 表。

延伸学习：

- [RFC 826: An Ethernet Address Resolution Protocol](https://www.rfc-editor.org/rfc/rfc826)  
  为什么推荐：这是 ARP 的经典标准文档，能让你知道 ARP 的本质是把协议地址转换成以太网地址。  
  今天读到什么程度：只读标题、Abstract 和开头 Problem 部分，不需要看完整字段格式。
- [Wireshark Display Filter Reference: ARP](https://www.wireshark.org/docs/dfref/a/arp.html)  
  为什么推荐：后面抓包会用 Wireshark 观察 `arp`，这个页面能确认 ARP 字段名和过滤器来源可靠。  
  今天读到什么程度：只知道 Wireshark 可以用 `arp` 过滤 ARP 包即可。

## 5. 访问外网时，ARP 找的是默认网关，不是网站服务器

这是今天最重要的点。

假设你的电脑配置如下：

```text
IP 地址：192.168.1.10/24
默认网关：192.168.1.1
DNS：192.168.1.1 或上游 DNS
```

你访问：

```text
www.example.com
```

过程不是：

```text
电脑 ARP 查询 www.example.com 的 MAC
电脑把以太网帧直接发给网站服务器 MAC
```

更准确的是：

```text
1. DNS：电脑先把 www.example.com 查成网站 IP，比如 93.184.216.34
2. 本地判断：93.184.216.34 不在 192.168.1.0/24
3. 路由选择：电脑决定交给默认网关 192.168.1.1
4. ARP：电脑查询 192.168.1.1 的 MAC
5. 发送帧：电脑把目的 MAC 写成 CPE/网关的 MAC，目的 IP 仍是网站 IP
```

这里有两个容易混淆的地址：

```text
以太网帧目的 MAC：默认网关的 MAC
IP 包目的 IP：网站服务器的 IP
```

家庭生活类比：

你寄外省快递时，信封最终收件人还是外省朋友；但你第一步要把包裹交给小区门口快递点。快递点不是最终收件人，但它是你本地第一跳。

工程意义：

- DNS 解决“网站名字对应哪个 IP”。
- ARP 解决“本地下一跳 IP 对应哪个 MAC”。
- 默认网关错了，DNS 查到网站 IP 也不一定能出网。
- ARP 到不了网关，终端连本地第一跳都交不出去。

延伸学习：

- [Cloudflare: What is DNS?](https://www.cloudflare.com/learning/dns/what-is-dns/)  
  为什么推荐：用较短篇幅解释 DNS 是域名到 IP 的解析系统，适合复习 Day002 的薄弱点。  
  今天读到什么程度：只复习 “What are the steps in a DNS lookup?”，重点区分 DNS 和 ARP。

## 6. 把 Day002 的 DNS 混淆修正到位

你在 Day002 已经能说清大部分路径，但 DNS 还有一个容易混淆点：

```text
DHCP 给终端 DNS 服务器地址；
DNS 查询返回网站 IP；
ARP 查询返回本地下一跳 MAC。
```

不要说：

```text
DHCP 给了网站 IP。
ONT 去查询 DNS。
ARP 查询网站域名。
```

更准确说：

```text
终端通过 DHCP 得到本地配置，其中可能包括 DNS 服务器地址。
打开网站时，终端按 DNS 配置查询域名，得到网站 IP。
如果网站 IP 不在本地网段，终端把包交给默认网关。
为了交给默认网关，终端用 ARP 查询网关 MAC。
```

## 7. 固网工程里为什么要学这些基础

在运营商固网里，很多大问题最后都会落到基础判断：

```text
用户有没有拿到 IP？
默认网关是谁？
能不能 ARP 到网关？
MAC 地址是否学习在正确端口？
用户在哪个 VLAN？
ONT/OLT 侧有没有在线和光功率证据？
BNG/BRAS 上有没有用户会话？
DNS 是否真的解析失败？
```

如果你把 DNS、ARP、默认网关、ONT、BNG 混在一起，排障会很快跑偏。

如果你能先分清：

```text
名字解析：DNS
本地下一跳 MAC：ARP
本地二层转发：交换/MAC
出家庭网络：默认网关/CPE
光接入：ONT/OLT
用户会话和策略：BNG/BRAS
```

后面的 VLAN、PPPoE、IPoE、DHCP、QinQ、BRAS 排障就会容易很多。
